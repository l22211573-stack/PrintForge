Roles y responsabilidades

1. Axel Vallejo – Arquitecto Backend y Seguridad

Lógica de Negocio Híbrida:Axel no solo hizo un carrito, programó un "validador de tipos". Si el carrito detecta un producto digital, el backend anula el costo de envío y genera un token de descarga. Si es físico, activa la solicitud de logística.
Capa de Seguridad: Implementó `password_hash()` con el algoritmo **BCRYPT**, asegurando que incluso si la base de datos se filtrara, las contraseñas serían ilegibles. También gestionó la "persistencia de sesión" para evitar que los usuarios pierdan su carrito al recargar.

2. Francisco Maldonado – DBA (Administrador de Base de Datos)

Normalización de Datos: Francisco diseñó la base de datos en **3ra Forma Normal**, evitando la duplicidad de datos. Creó llaves foráneas (`Foreign Keys`) para asegurar la "Integridad Referencial" (que no existan pedidos de usuarios que no existen).
Automatización con Triggers: Desarrolló scripts que descuentan automáticamente el stock de la tabla `productos` cuando se inserta una nueva fila en `detalle_pedido`, manteniendo el inventario sincronizado sin intervención manual.

3. Ernesto Rivera – Lead Frontend y UX/UI

Interfaz Dinámica (SPA Concept): Ernesto eliminó las recargas de página innecesarias usando **Fetch API** y **Promesas de JavaScript**. Los productos se inyectan en el DOM en tiempo real, creando una experiencia de "Single Page Application".
Diseño Adaptativo: Implementó un sistema de rejillas (**Grid/Flexbox**) para que la tienda sea 100% funcional en dispositivos móviles, permitiendo que un "maker" pueda comprar refacciones desde su celular mientras repara su impresora.

4. Jesús Lizárraga (Chuy) – Ingeniero de Hardware e IoT

Middleware de Comunicación: Jesús configuró la Raspberry Pi Pico W** para actuar como un cliente HTTP que hace "polling" (consultas constantes) al servidor central.
Interacción Máquina-Humano: Programó la lógica física; cuando el backend registra una venta, la Pico W interpreta la señal y activa indicadores (LEDs o pantallas) que notifican al operador en el taller que debe preparar un paquete, conectando el software con el mundo real.

5. Ángeles González – QA (Control de Calidad) y Documentación

Protocolo de Pruebas (Beta Testing): Ángeles ejecutó pruebas de "caja negra", intentando romper el sistema (por ejemplo, comprando productos sin stock o con precios negativos) para asegurar que el backend de Axel tuviera las validaciones correctas.
Technical Writing: Tradujo el lenguaje técnico de los programadores a un **Manual de Usuario** y un **Manual de Instalación** claros. Ella es quien garantiza que el profesor pueda clonar el repositorio y hacerlo funcionar en 5 minutos siguiendo las instrucciones.
